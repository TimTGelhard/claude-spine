The spine of every
Claude Code project.

Installable skills, atomic discipline chapters, and personalization for operators already shipping with Claude Code.

Not a tutorial. Not a marketplace. The toolbox a senior dev would build for themselves, made install-shareable.

Install

git clone https://github.com/TimTGelhard/claude-spine ~/.claude-spine
cd ~/.claude-spine
./install.sh

macOS or Linux (Windows via WSL). Requires bash, jq, git. Existing ~/.claude/ files are backed up before any change. Re-running is safe.

What you get

  • 21 op-* skills

    Each routes Claude to the right atomic chapter on demand — never the whole folder. Descriptions are the trigger; bodies stay small. Loaded only when relevant.

  • ~80 atomic chapters

    One concept per file, <150 lines each. Foundations, workflow, prompting, signaling, persistence, tools, subagents, recovery, anti-patterns. Indexed and discoverable.

  • Personalization layer

    /onboard writes a profile that calibrates the entire spine — your stack, push-back intensity, output style. Plus a bucket loop that grows your personal library one curated addition at a time.

Single-maintainer, opinionated repo. Fork it freely; upstream PRs welcome for factual errors and universal patterns.
